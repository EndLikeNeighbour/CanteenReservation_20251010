# 天津宴餐厅预订系统

## 📋 项目地址
<https://github.com/EndLikeNeighbour/CanteenReservation_20251010>

## 📋 项目描述

本项目是南开大学"面向对象程序设计"课程的期末大作业，旨在开发一个功能完善的餐厅预订管理系统，替代传统的手工预订方式。系统基于C++语言开发，采用Qt框架构建图形用户界面，实现了顾客、员工和经理三类用户的协同操作。

## 🎯 项目背景

随着餐饮业的快速发展，传统手工预订系统已无法满足现代餐厅的运营需求。某餐厅目前使用手工预订表单记录餐桌预订信息，存在效率低下、易出错、难以识读、难以获取管理数据等问题。因此，开发一个预订管理系统成为当务之急。
对比市场上已有的餐厅管理系统，大多功能复杂、成本高昂，且不符合该餐厅的操作习惯。因此，需要开发一个简洁、易用、符合餐厅实际需求的定制化系统。

## ✨ 主要功能

### 顾客端功能
- **注册与登录**：通过手机号识别新老顾客
- **菜单查看**：查看餐厅完整菜单（按类别分组）
- **餐桌预订**：选择时间、人数、特殊需求进行预订
- **预订管理**：查看个人所有预订记录、取消预订

### 员工端功能
- **状态监控**：查看所有餐桌实时状态
- **预订处理**：确认/拒绝顾客预订
- **顾客管理**：标记顾客到店/离开
- **临时处理**：处理临时到店顾客（无需预订）

### 经理端功能
- **资源管理**：添加/移除餐桌、管理菜单（添加/修改菜品）
- **业务分析**：生成预订报表（按日期导出为txt文件）、查看业务统计
- **员工功能集成**：具备员工所有功能

## 🏗️ 系统架构

### 三层架构设计
- **业务层**：核心业务逻辑类（Customer、Staff、Manager、Restaurant）
- **数据层**：基础数据结构与核心业务类（Table、Reservation、Menu）
- **表示层**：基于Qt的图形用户界面

### 开发环境
- **编程语言**：C++ 17
- **GUI框架**：Qt 5.14.2
- **设计模式**：单例模式、三层架构

## 🚀 安装与使用

### 编译步骤
#### 使用Qt Creator
1. 打开Qt Creator
2. 导入项目文件（.pro）
3. 配置构建套件
4. 构建并运行

### 运行说明
1. 启动程序后进入主界面
2. 选择用户类型登录：
   - 顾客：输入手机号（新顾客需补充姓名）
   - 员工：输入员工工号（任意非空字符串）
   - 经理：输入经理工号（以M开头）
3. 根据界面提示进行相应操作
